## 3. The shared contract — designed for sync (lock this first)
Add/confirm in **`backend/shared/types/src/index.ts`** (the single source both apps import):

- **`Mode`** — the batch/trial delivery mode. ⚠️ **Three spellings exist today and must be reconciled:** prisma **`BatchMode = online | offline`** (where `offline` **≡ "in-studio"**), shared-types `mode?: 'in-studio' | 'online'` (Batch, ~line 200) and `'online' | 'offline'` (Workshop, ~line 166), and the master docs' **"In-studio / Online"** label. Pick ONE canonical stored value set + a display-label mapping and use it everywhere (see §8 decision 1).
- **`Plan`** — `monthly | quarterly | annual`. (`biannual` is being dropped — see S0.2/S2.3; do **not** add it.)
- **`AttendanceMode`** — `otp | qr | auto_join` (which mechanism a booking uses: trial=otp, in-studio=qr, online=auto_join).
- **Money** — all amounts are integer **paise** (`*Paise` fields already used, e.g. `monthlyFeePaise`). Document it.
- **Discount** — **bps**, hard cap **3000 (30%)**. Reserve field names `quarterlyDiscountBps` / `annualDiscountBps` (added in S1.3).
- **Error envelope** — already `{ error: { code, message, fields? } }` via `AppError(code, status, message, fields)`. Lock the **code catalog** as a shared union (e.g. `VALIDATION | UNAUTHORIZED | FORBIDDEN | NOT_FOUND | CONFLICT | RATE_LIMITED | PAYMENT_FAILED | …`) so clients can switch on `code`.
- **Pagination** — confirm the existing cursor shape is exported and reused.

> These are *additions/locks* to an existing file — keep the current exports intact; both apps already import from here.

## 8. Open decisions (resolve here)
1. **Mode value set (blocking for S1.3).** Today: prisma **`BatchMode = online | offline`** (`offline` = in-studio), shared-types `'in-studio' | 'online'`, docs say **"In-studio / Online"**. Decide the canonical *stored* value + the *display* label:
   - **Option A (no migration):** keep stored `online | offline`; render `offline → "In-studio"` in both apps via the shared kit. Cheapest.
   - **Option B (clarity):** migrate `BatchMode → in_studio | online` to match product language; update all references + a DB migration.
   - **Recommendation:** Option A for speed (single label-map in `backend/shared`), revisit if `offline`-means-in-studio causes confusion. **Lock before S1.3** (which reads batch mode everywhere). Also reconcile Workshop's separate `online | offline` (there, `offline` genuinely means in-person) so the two `Mode` uses don't get conflated.
2. **Error-code catalog** — enumerate now from existing `AppError` throw-sites (`grep "new AppError(" backend/api/src`) so it's exhaustive.
3. **`fitness` category** — add to prisma `Category` + `CategoryEnum` now or later (affects S0.4/S1.2).
