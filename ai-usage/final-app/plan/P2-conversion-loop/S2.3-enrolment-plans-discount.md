## 1. Feature & intent
Enrol into a batch on **Monthly / Quarterly / Annual**, priced **from the batch's academy-set discount** (`quarterlyDiscountBps`/`annualDiscountBps`, S1.3) вЂ” not hardcoded percentages. Drop `biannual`, remove promo codes, and surface a real plan picker (Quarterly *Popular* / Annual *Best value*). Manage lifecycle (renew/pause/transfer/discontinue) is REUSE.

- **Refs:** master-feature-document В§3.5 (enrol + plans) / В§2.5 (academy-set discounts); prototypes enroll selectв†’payв†’confirm + manage; gap-analysis (enrol screen lacked a real picker + used promo codes; renewal "already reads discount_bps" вЂ” **disproved below**).
- **Post-P1 reality (verified):** enrol is **`POST /batches/:id/enroll { package_type }`** (`batches/routes.ts:16`), **already server-authoritative** (client sends only the plan). But `computeAmount()` (`enrollments/service.ts:35`) reads **hardcoded `PACKAGE_CONFIG`** (`:12` вЂ” monthly 0 / quarterly 500 / biannual 1000 / annual 1500 bps), **not** the batch fields. **Renewal has the identical bug** (`renewEnrollment` + `getRenewalOptions` use the same `PACKAGE_CONFIG`). `PlanEnum` in `@findemy/types` is **already** `monthly|quarterly|annual` (no biannual) вЂ” but `biannual` is still live in 5 runtime places.

## 3. The shared contract (lock first)
```
POST /batches/:id/enroll { package_type: 'monthly' | 'quarterly' | 'annual' }
  в†’ server prices from the BATCH:
      months   = { monthly:1, quarterly:3, annual:12 }[plan]
      discountBps = plan==='quarterly' ? batch.quarterlyDiscountBps
                  : plan==='annual'    ? batch.annualDiscountBps
                  : 0
      amount_paise = gross в€’ floor(gross * discountBps / 10000),  gross = monthlyFeePaise * months
  в†’ returns { enrollment_id, amount_paise, ... }  (NO client-sent amounts)
GET /batches/:id/renewal-options в†’ same pricing, per plan, with discount_bps echoed for the UI
```
- **Drop `biannual` everywhere in one pass** (it's not in `PlanEnum`, so the type already forbids it вЂ” but runtime code/UI still offer it). After removal, the plan set is exactly `monthly|quarterly|annual`.
- **Discount math is REUSE** (`computeAmount` already does `gross в€’ floor(gross*bps/10000)`); only the **bps source** changes (hardcoded в†’ batch). The batch is fetched with **`findUnique`/`include:{batch:true}` (no `select`)**, so `quarterlyDiscountBps`/`annualDiscountBps` are **already loaded** вЂ” no query change needed, just **read them** instead of `PACKAGE_CONFIG`.
- **No promo codes** in MVP (decided): remove the client-side `FIRST50`/`SAVE200` UI (no backend promo endpoint exists вЂ” confirmed).
