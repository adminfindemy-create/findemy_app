# S3.4 — Schedule + cancel-class + make-good `[F11, F8]` — CHANGE / BUILD

> Feature Master Plan (master-execution-plan §3.2). Deps: **S2.3** (enrolment). Also **finishes the slot→timing transition** the academy schedule still depends on (overlaps DF1).

## 1. Feature & intent
The academy schedule (Now / Upcoming / Cancelled + Online). **Cancelling a class** marks it **unbilled** and appends **one make-good session** to each affected active enrolment so the entitled count is preserved (cancel one of an 8-class month → student still gets **8**); the student is **notified**. Reschedule was already discarded (S0.2).

- **Refs:** master-feature-document §2.5 (cancel-class + make-good) / §4.3 (Schedule); prototypes Schedule + cancel-class sheet ("unbilled · +1 make-good · count preserved").
- **Post-P2 reality (verified):** the academy schedule **still reads dormant `Slot` rows** (`studio/repo.ts getSchedule:153`) and cancels via a **raw `DELETE /studio/slots/:id` → hard delete** (`slots/service.ts:65`, `repo.ts:39`) — **no record, no make-good, no credit**. `ScheduleItem` (`@findemy/types`) still carries `slot_id`. **No `SessionCredit`/`CancelledSession` model; the `Batch` has no session-count** and `EnrollmentPeriod` tracks entitlement implicitly via `endDate`. `push/service.ts sendPushNotifications` + `enrollments/repo.ts findEnrollmentsForBatch` are REUSE.

## 3. The shared contract (lock first)
```
GET /studio/schedule?week_start= → days[].items[]: ScheduleItem {
  batch_id, session_date(IST), start_time, end_time, mode: Mode,
  status: 'now'|'upcoming'|'done'|'cancelled', enrolled_count, capacity, coach_name
}   // projected from BatchTiming; cancelled overlaid from CancelledSession. NO slot_id.

POST /studio/sessions/cancel { batch_id, session_date }   (academy auth)
  → validate: batch is the academy's, session_date is a real BatchTiming occurrence, and it is strictly FUTURE (not started/past)
  → CancelledSession: findUnique((batchId,sessionDate)); if it exists → return (already cancelled, no-op).
    Else create it, and ONLY THEN (newly-created branch) — all in one $transaction:
       for each affected enrolment (active, NOT paused, with a current active period — see ⚠️):
        (a) create SessionCredit (+1, reason 'class_cancelled')   — audit / count
        (b) EXTEND its period: extendPeriodEndDate(period, nextOccurrenceAfter(period.endDate), grace+Δ)  — DELIVERS one session
  → notify each affected student
  → returns { cancelled, affected_count }

Batch.sessionsPerMonth: Int   // academy-set at batch creation (owner decision) — the explicit class count
CancelledSession { id, batchId, sessionDate(Date), reason?, cancelledByAccountId, createdAt, @@unique([batchId, sessionDate]) }
SessionCredit    { id, enrollmentId, batchId, cancelledSessionId?, reason, sessionCount @default(1), createdAt }
entitled_sessions = Batch.sessionsPerMonth × period-months + Σ SessionCredit.sessionCount   // "8 stays 8" — now an explicit, displayable count
```
- **Cancellation creates a record, not a deletion** — there's no `Slot`/session row to delete (D1). `CancelledSession` is the unbilled marker + the audit trail + the idempotency key.
- **Explicit session count (owner decision).** `Batch.sessionsPerMonth` (set by the academy alongside days/times in batch creation) gives a real entitled count — `sessionsPerMonth × period-months` — so "8 stays 8" is a concrete, displayable number, not a guess.
- **Make-good is DELIVERED *and* counted.** On cancel: **(a)** record a `SessionCredit` (+1 — the audit + the `+ Σ credits` in the count) **and (b)** extend the affected period so the student physically gets the class back. Together: the displayed count stays 8, *and* an extra session occurs.
- **⚠️ Extend to the NEXT class occurrence, not "+7 days" (verified flaw).** A flat interval is wrong: a 2×/week batch (e.g. Sat+Sun) has a ~1-day inter-session gap, so `endDate + 7d` could add **0 or 2** sessions depending on which day was cancelled. Instead compute **`nextOccurrenceAfter(period.endDate)`** — the first `BatchTiming` occurrence strictly after the current `endDate` — and `extendPeriodEndDate(period, that, grace shifted by the same Δ)`. That adds **exactly one** session. (`projectTrialTimes` (`batches/service.ts`) projects from `now`; generalize it to project from an arbitrary start, or write a small `nextOccurrenceAfter(date, timings)`.)
- **⚠️ Recipients = active, NOT paused, with a current active period (verified gaps).** `findEnrollmentsForBatch` (`enrollments/repo.ts:210`) only filters **periods** (`where:{ batchId }`, no enrolment filter), so it returns discontinued **and paused** enrolments. After the call, keep only `status==='active' && (pausedUntil==null || pausedUntil<today) && hasActivePeriod` — else a discontinued or paused student gets a spurious make-good. (`Enrollment.pausedUntil` exists, `schema.prisma:534`.)
- **⚠️ Idempotency — gate on newly-created (verified nuance).** Prisma `upsert` returns the row but **does not signal create-vs-update**, so it can't gate the side effects. Use **`findUnique` → if present, no-op; else `create` then credit/extend** (or `create` + catch P2002), all inside the `$transaction`. Cancelling the same session twice → one `CancelledSession`, **no double credit / double extension**.

1. Backend: `CancelledSession`+`SessionCredit` (db push); rewrite `getSchedule` off timings; cancel endpoint + credits + notify; remove `deleteSlot` → handoff. 2. Academy schedule binds + cancel sheet. 3. E2E.

**Definition of Done (E2E)**
- [ ] Academy cancels one session of an 8-class month → each active enrolment gets a `SessionCredit` **and its period `endDate` extends one interval** (the make-good is delivered, not just logged); the session is **unbilled** (recorded); a **notification fires**.
- [ ] Cancelling the same session twice → **no double credit / no double extension** (guarded by the unique `CancelledSession`).
- [ ] A **discontinued** enrolment on that batch is **not** credited.
- [ ] The schedule renders from **`BatchTiming`** (no `Slot` query); cancelled sessions show as **Cancelled**; online sessions show the mode.
- [ ] The raw `DELETE /studio/slots/:id` is gone. Typecheck + full suite green; `db push` clean.
