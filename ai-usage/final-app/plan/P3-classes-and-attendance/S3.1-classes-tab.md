## 3. The shared contract (lock first)
```
GET /me/classes → {
  active: ClassItem[],   // enrolment status active (current period active/grace)
  past:   ClassItem[],   // enrolment inactive OR latest period expired/cancelled
}
ClassItem {
  enrollment_id, batch_id, batch_title, category, level, academy_id, academy_name,
  coach_name, mode: Mode ('in-studio'|'online'), monthly_fee_paise,
  status: 'active'|'paused'|'past', started_at, current_period_end?,
  attended_count,   // count of BatchAttendance present rows for (user, batch)
  timings: BatchTiming[],
}
```
- **Enrolled only — trials are NOT classes.** Trials live in the booking flow (`Booking`→`Trial`) and stay in **Profile → Bookings**. The Classes read is purely over `Enrollment`/`EnrollmentPeriod`.
- **Active vs past:** *active* = `Enrollment.status='active'` with a current period `active`/`grace`; *past* = `Enrollment.status='inactive'` OR the latest period is `expired`/`cancelled`. Paused enrolments are *active* with a `paused` flag.
- **`mode` drives the in-class action** (scan vs join) — already canonical from S1.3.

## 4. Component design, per side
- **Backend:** add `getMyClasses(userId)` (service) + `findMyClasses` (repo) splitting active/past; reuse the existing enrolment mapping. Route `GET /me/classes` (student auth). **Fix `attended_count`** → `prisma.batchAttendance.count({ where: { userId, batchId, present: true } })` (the real class number; trials have their own count in Bookings). Add the route to `@findemy/types` + api-client (`me.getClasses`).
- **Student app:** `(tabs)/classes.tsx` (new) + add the tab to `_layout.tsx` (grad-cap icon; 4 tabs Discover·Events·Classes·Profile). **Adaptive states** from the `{active, past}` payload:
  - `active.length > 1` → **cards**; `active.length === 1` → **inline single**; `active.length === 0 && past.length > 0` → "not in any active batch" + **past cards**; `active+past == 0` → **tab hidden** (don't render the tab). **⚠️ Tab visibility needs `active+past` before the screen mounts** — `getMyStats.enrolled_count` is active-only, so either extend `getMyStats` with a `has_classes` flag (`active+past>0`) or have `_layout.tsx` read `GET /me/classes` to decide. Don't gate on the active-only count (a past-only student would wrongly lose the tab).
  - Repurpose `enrollments.tsx` card UI into the tab; keep `enrollment/[id].tsx` as the detail (manage sheet REUSE). Move the Profile "Your Classes" quick-link to the tab; "Past classes" stays reachable.
- **Academy app:** none.
- **Shared:** `ClassItem`/`MyClassesResponse` types in `@findemy/types`.
