# S1.1 — Academy profile & media gallery `[F4, F21]` — REUSE / CHANGE

> Feature Master Plan (master-execution-plan §3.2). Deps: S0.1. **The gallery already exists** — this slice is mostly **add video** + surface the now-existing `modesOffered`.

## 1. Feature & intent
A rich academy profile both apps render consistently: cover + **media gallery of up to 10 photos *and videos***, name/rating/location, about, coaches, reviews preview, and the academy's **offered modes**. The profile editor keeps bank/payout details (decided §4).

- **Refs:** master-feature-document §3.3 (academy detail) / §4.11 (Studio + Edit profile); prototypes academy detail + edit-profile; gap-analysis §3 (gallery ≤10 already built, **add video**; keep bank details) + §4 decisions 3 & 4.
- **Post-P0 reality:** gallery built (cap 10, image-only); `modesOffered` added in S0.3 (returned as `modes_offered`); bank fields present on `Academy`.

## 3. The shared contract (lock first)
- **Two reads need `modes_offered` added** (verified — *neither* returns it today, though both fetch the full `Academy` row):
  - **Student-facing** `GET /academies/:id` (`academies/service.ts`) — for the student academy detail.
  - **Academy-facing** `GET /studio/academy` (studio module → `useStudioAcademy`) — for the academy to see its *own* offered modes on Studio/profile.
  - Both are **one-line mappings** (`modes_offered: academy.modesOffered`), no query change.
- Media item type: keep `images: string[]` (URLs); a video URL is just another entry. **Decision:** keep a flat `string[]` and detect type by extension/MIME at render, OR add a typed `media: { url, kind: 'image'|'video' }[]`. Recommendation: **flat `string[]` + render-time detection** (smallest change; matches the existing cap-10 array). Document in the handoff.
- Upload contract (verified route names): **`POST /studio/academy/upload`** (multipart) — **accept image + video MIME**, keep ≤10 total; **`DELETE /studio/academy/image`** by URL (REUSE).

## 4. Component design, per side
- **Backend:** (1) `studio/upload.ts` — extend the allowed MIME set (e.g. `video/mp4`, `video/quicktime`) + ext mapping (`:31`); keep the `>= 10` cap; store the URL in `Academy.images` (REUSE); consider a max file-size guard for video (R2). (2) **add `modes_offered: …modesOffered`** to **both** the public academy serializer (`academies/service.ts` detail; + list if cards surface modes) **and** the academy's own read (`GET /studio/academy`, studio module) — neither returns it today.
- **Academy app:** `profile/edit.tsx` — `ImagePicker.MediaTypeOptions.All`; thumbnail videos in the picker grid; the `uploadMultipart` flow is REUSE. Add a read-only **offered-modes** display (from `modes_offered`).
- **Student app:** `academy/[id].tsx` — the gallery FlatList renders a `<Video>`/`expo-video` for video URLs (poster + tap-to-play), `<Image>` for photos. Add offered-modes chips near the header.
- **Shared:** if adopting typed media, add `MediaItem` to `@findemy/types`; otherwise none.
