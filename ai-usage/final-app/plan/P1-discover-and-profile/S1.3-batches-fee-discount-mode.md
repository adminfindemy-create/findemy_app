# S1.3 — Batches: fee + discount + mode (catalog source of truth) `[F10, F4]` — CHANGE / BUILD

> Feature Master Plan (master-execution-plan §3.2). Deps: S0.1, S0.4. **This is the P1 critical path** — it defines the contract S2.1 (trial spots), S2.3 (enrol pricing), and S3.x (attendance mode) all read. Do it last in P1.

## 1. Feature & intent
Make the **`Batch` the single source of truth** for: **monthly fee**, **academy-set quarterly/annual discount %** (≤30%), **delivery mode**, and **capacity** → from which **trial spots = capacity − enrolled** is computed. Today batches have fee + mode + capacity but **no discount fields**, and the batch read serializes mode raw.

- **Refs:** master-feature-document §2.4 (trial spots) / §2.5 (academy-set discounts, ≤30%) / §4.5 (New batch + batch detail); prototypes new-batch sheet + batch detail; gap-analysis §1/§3 (discount fields + ≤30% cap missing; new-batch lacks discount).
- **Post-P0 reality:** `Batch` has `capacity/trialFeePaise/monthlyFeePaise/mode(BatchMode default offline)/status` — **no `*DiscountBps`**. `BatchCreateSchema`/`BatchUpdateSchema` (studio routes) lack discount. `getBatchAvailability` **already returns `available = capacity − enrolled`**. `biannual` still lives in `validPackages`/`PACKAGE_CONFIG` (deferred to S2.3).

## 3. The shared contract — designed for sync (lock first)
```
Batch {
  …existing: capacity, trialFeePaise, monthlyFeePaise, mode, status,
  quarterlyDiscountBps: int (0–3000),   // NEW, academy-set, ≤30%
  annualDiscountBps:    int (0–3000),   // NEW
  trialSpots: int,                       // computed = capacity − enrolled (read-only)
  mode: Mode,                            // wire = 'in-studio' | 'online' (S0.1); DB stays online|offline
}
POST/PATCH /studio/batches { …, quarterly_discount_bps?, annual_discount_bps? }  // reject > 3000 → VALIDATION
```
- **Money rail:** fees in paise, discounts in **bps capped 3000** (S0.1).
- **Mode rail (bidirectional — don't half-do it):** stored `online|offline`; **wire = canonical `'in-studio'|'online'`** (Option A). Map at the API boundary **both ways**: read serializers `offline → 'in-studio'` (mirror `studio/service.ts:93`); **and the write path** — `BatchCreate/UpdateSchema.mode` currently takes `z.enum(['online','offline'])`, so either canonicalize it to `'in-studio'|'online'` (map back to DB) **or** keep writes raw and have the apps map. ⚠️ **Normalizing reads alone breaks the academy editor:** `batches/[id]/edit.tsx:48` does `setMode(existing.mode ...)` into a toggle typed `'online'|'offline'` — a canonical read value (`'in-studio'`) would mismatch its type + toggle logic. **Recommendation:** canonicalize the whole batch-mode vocabulary (read **and** write) to `'in-studio'|'online'` and update the academy new/edit toggles to match, so the app uses one spelling end-to-end.
- ⚠️ **Discount is *stored* here but not yet *consumed*.** Enrol pricing still uses hardcoded `PACKAGE_CONFIG` until **S2.3** rewires it to read `Batch.*DiscountBps` (and drops `biannual`). S1.3 establishes the source of truth ahead of that.

  - batch service/repo: persist discounts; **batch read serializers** → add `trialSpots` (reuse `getBatchAvailability` logic) + map `mode` to canonical `Mode` (`offline → 'in-studio'`). ⚠️ **Normalize ALL batch-mode read emission points** (verified — three serialize raw today): `academies/service.ts:141`, `studio/service.ts:127` & `:151`, and **`auth/service.ts:649` (student enrollments / "My Classes")**. (The trial mapping at `studio/service.ts:93` is already correct.) Update `@findemy/types` `Batch.mode` to `Mode` and **remove the `TODO(S1.3)`**.
- **Academy app:** `batches/new.tsx` + `batches/[id]/edit.tsx` (both exist; hooks `useCreateBatch` / `useUpdateBatch(id)` exist) — add **quarterly %** + **annual %** inputs (0–30, block >30 with inline error); **update the mode toggle** (currently typed `'online'|'offline'`, and `edit.tsx:48` reads `existing.mode`) to the canonical mode vocabulary chosen in §3 so read/write stay consistent; batch detail header → **Online badge** + **"N trial spots open"** + the "trial spots open automatically (capacity − enrolled)" note. `useCreateBatch`/`useUpdateBatch` payloads carry the discount bps.
