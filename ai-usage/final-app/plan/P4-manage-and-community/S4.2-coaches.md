# S4.2 — Coaches `[F17]` — REUSE (+ small CHANGE)

> Master §P4. Deps: **S1.3** (batch model). Coaches are structurally aligned; this slice surfaces a coach's batch-count and a post-creation assign-to-batch affordance.

## 1. Feature & intent
The academy manages **coaches** (list / detail / add) and **assigns a coach to a batch**; the coach drives the program/batch display the student sees.

- **Refs:** master §4.7; gap-analysis (structurally aligned).
- **Post-P3 reality (verified):** `coaches/*` CRUD is complete — `GET/POST /studio/coaches`, `PUT/DELETE /studio/coaches/:id` (`coaches/routes.ts:20-38`); `Coach` has `batches: Batch[]` via `Batch.coachId` (`schema.prisma:150-164`). Coach name **already shows** on the student batch hub (`batches/[id]/index.tsx:82`) + academy batch cards. **Gaps:** the coach read returns no **`batch_count`** (`coaches/service.ts:6-13`); assignment happens **only at batch create/edit** (`Batch.coachId`) — there's **no assign affordance on the coach screens**, and coach detail (`coaches/[id].tsx`) doesn't list its batches.

## 2. Cross-side impact map
| | Backend (`backend/api`) | Academy app |
|---|---|---|
| Batch-count | **CHANGE** coach read → derive `batch_count` from `Batch` assignments (`_count.batches`) | **CHANGE** coach list/detail show "N batches" |
| Coach's batches | **CHANGE** coach detail read → include the assigned batches (id, title) | **CHANGE** `coaches/[id].tsx` lists assigned batches |
| Assign to batch | **REUSE** `PUT /studio/batches/:id { coach_id }` (already exists) | **BUILD** an "Assign to batch" affordance on coach detail → picks one of the academy's batches → calls the existing batch update |
| Student display | REUSE (coach name already on program/batch) | — |

## 3. The shared contract (lock first)
```
GET /studio/coaches → { items:[{ id, name, specialty, bio?, phone?, batch_count }] }   // + batch_count
GET /studio/coaches/:id → { coach:{ …, batches:[{ id, title }] } }                       // NEW route (doesn't exist today)
PUT /studio/batches/:id { coach_id }   // REUSE — the assign path (no new endpoint) — see ⚠️
```
- **No new *assignment* endpoint** — assignment is a batch property (`Batch.coachId`). The coach-detail "assign" affordance calls `PUT /studio/batches/:id` with the chosen `coach_id` (reuse `useUpdateBatch`). Keeps one source of truth.
- **⚠️ Validate coach ownership on assign (verified gap).** `updateBatch` (`studio/service.ts:213-245`) validates the *batch* is the academy's but **does NOT check the `coach_id` belongs to the academy** — so a client could assign another academy's coach. Add: when `coach_id` is provided, verify `Coach(id=coach_id, academyId)` exists (404 otherwise). This is a real fix to the reused path, not just S4.2 wiring.
- **`batch_count`** = `prisma.coach … _count: { select: { batches: true } }` (or filter to active batches). Derived, never stored.

## 4. Component design, per side
- **Backend:** `coaches/repo.ts` list → add `_count: { batches: true }`; `coaches/service.ts` → map `batch_count`. **BUILD `GET /studio/coaches/:id`** (no detail route exists today — `coaches/routes.ts` has list/create/update/delete only) → `getCoach(id, academyId)` with `include: { batches: { select: { id, title } } }`. Add the coach-ownership check to `updateBatch` (the assign path).
- **Academy:** `coaches/index.tsx` rows show "N batches"; `coaches/[id].tsx` lists assigned batches + an **"Assign to a batch"** row → a small batch picker (the academy's batches) → `useUpdateBatch(batchId).mutate({ coach_id })` → invalidate coaches + batches. `new.tsx` unchanged.
