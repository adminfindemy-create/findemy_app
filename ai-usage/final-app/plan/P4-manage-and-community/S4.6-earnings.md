## 3. The shared contract (lock first)
```
GET /studio/earnings?period=week|month|year
  ‚Üí { gross_paise, commission_paise, net_paise,
       by_category:[{ kind, gross_paise, commission_paise, net_paise }],
       by_batch:[‚Ä¶], transactions:[{ ‚Ä¶, gross_paise, commission_paise, net_paise }],
       payouts:[‚Ä¶] }

commission for a window = ő£ (commissionPaise ‚ąí reversedPaise)
   from CommissionLedger WHERE academyId=? AND status='accrued' AND createdAt ‚ąą window
net = gross ‚ąí commission
```
- **Join on `(itemType, itemId)`** to attribute commission to each payment line ‚ÄĒ **trial‚Üí`bookingId`, enrolment‚Üí`enrollmentPeriodId`, workshop‚Üí`workshopPaymentId`** (events carry **no commission** ‚ÄĒ platform-owned, S4.5 ‚ÄĒ so their lines are gross == net), **or** aggregate by `academyId` for the period total. Use the per-item join for the transactions list, the aggregate for the headline.
- **`commissionPaise ‚ąí reversedPaise`** is the live commission (refund reversals reduce it). Only `status` rows that have accrued count; fully-reversed rows net to ~0.
- **Non-breaking shape:** add the three `*_paise` fields alongside the existing ones so nothing else breaks; the UI swaps `total √ó 0.85` for `net_paise`.
- **Window basis:** commission `createdAt` ‚Čą the payment `capturedAt` (commission is written in the capture txn) ‚ÄĒ same window as gross, so they line up.
