# S4.4 — Workshops (Online/Offline, non-refundable) `[F18]` — CHANGE

> Master §P4. Deps: **S2.4** (payments + commission). Narrows the workshop type, removes the refund path, and **closes the commission-recording gap** (a verified bug) so earnings (S4.6) can net workshops.

## 1. Feature & intent
The academy hosts **workshops** (type **Online** or **Offline** only); a student registers + pays; registration is **non-refundable**. The workshop payment records Findemy's commission.

- **Refs:** master §3.8/§4.9; gap-analysis (trim masterclass/demo; remove refund path).
- **Post-P3 reality (verified):** `WorkshopType = online|offline|masterclass|demo` (`schema.prisma:450-458`). The register→pay flow is complete (`workshops/routes.ts:15-18,34-37`); `cancelWorkshopRegistration` has a **24h refund branch** (`workshops/service.ts:206-232`). Academy type grid has **4 options** (`workshops/new.tsx:31-36`) + `TYPE_LABELS` (`(tabs)/workshops.tsx:19-24`). **⚠️ Verified bug:** the workshop payment-capture handler (`payments/service.ts:188-211`) **does NOT call `recordCommission`** — unlike trial (`:233`) + enrolment (`:129`) — even though `CommissionItemType.workshop` + `recordCommission(workshop)` exist (`payments/repo.ts:218`). Student cancel UI (`CancelSheet.tsx`) is already refund-aware and **auto-corrects** once the backend stops returning `refund_eligible`.

## 3. The shared contract (lock first)
```
WorkshopType = online | offline                         // was online|offline|masterclass|demo
POST /workshops/:id/register → pay                       // REUSE
cancel: NON-REFUNDABLE, full stop                        // remove the >24h refund branch
payment.captured (workshop): recordCommission(workshop, itemId=workshopPaymentId, gross=amountPaise)  // NEW (parity w/ trial/enrol)
```
- **Non-refundable, full stop.** Delete the `refundEligible = isPaid && hoursUntilWorkshop > 24` branch + the refund-initiation block (`:206-232`); cancellation just sets the registration cancelled (frees the spot if you want) and returns `{ refund_initiated: false }`. The student `CancelSheet` already renders the no-refund card from that.
- **Commission on capture (parity).** Mirror the enrolment handler exactly: inside the workshop `payment.captured` transaction, after marking the `WorkshopPayment` captured, `recordCommission({ academyId: workshop.academyId, itemType:'workshop', itemId: workshopPayment.id, grossPaise: amountPaise }, tx)`. Idempotent via the ledger's `@@unique([itemType,itemId])`. **This is the line S4.6 depends on.**
