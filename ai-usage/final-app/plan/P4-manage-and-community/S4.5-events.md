# S4.5 — Events `[F19]` — REUSE (+ commission fix)

> Master §P4. Deps: **S2.4** (payments + commission). Events are structurally complete and Findemy-curated; the only real work is closing the same commission-recording gap workshops had.

## 1. Feature & intent
Students **browse + register** for events (free or paid); paid events run through the existing payment rails. Events are **Findemy-curated** (no academy authoring in MVP).

- **Refs:** master §3.9; app-ideation (the Events *module* is MVP; championships future).
- **Post-P3 reality (verified):** `events/*` is complete — browse/detail/register/pay/cancel (`events/routes.ts`), `Event`/`EventRegistration`/`EventPayment` models, `EventType = competition|talent_hunt|meetup`. Cancellation is **type-scoped** already (competition/talent_hunt non-refundable; meetup refundable >2h) — **no flat-24h branch to remove**. Student screens (`events/[id].tsx`, `events/pay.tsx`) are complete with correct type-specific copy. **No academy surface** (correct).
- **✅ RESOLVED (verified): events carry NO commission.** The `Event` model has **only `organizerName`, no `academyId`** (`schema.prisma:419-442`), and `recordCommission` requires a **non-null `academyId`** (`payments/repo.ts:218`). Events are **Findemy-curated, platform-owned** — there is no academy to attribute commission to. So **S4.5 records no commission and wires no reversal**; the platform simply keeps 100% of event revenue. The earlier "confirm ownership" question is closed: **platform-only, by schema.** This makes S4.5 a near-pure REUSE/confirm slice, and removes "events" from S4.6's commission netting.
