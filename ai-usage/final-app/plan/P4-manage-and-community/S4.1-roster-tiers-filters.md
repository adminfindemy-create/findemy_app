# S4.1 — Students roster, tiers & filters `[F16]` — CHANGE

> Master §P4. Deps: **S3.2/S3.3** (real attendance in `BatchAttendance`). The tier engine already exists server-side; this slice makes it consistent, surfaces it in the roster rows, and replaces the category chips with a Batch×Attendance Filters sheet.

## 1. Feature & intent
The academy's **Students** tab: each student shown with **avatar · batch · attendance % · tier** (Active ≥75 / Irregular 50–74 / Inactive <50), filterable by **Batch × Attendance** via a Filters sheet. Tiers derive from real class attendance (P3).

- **Refs:** master §4.4; prototypes Students + filters sheet; gap-analysis (`at_risk`→Irregular; rows lack %/tier; no Filters sheet; drop category chips).
- **Post-P3 reality (verified):** tier is computed in `studio/repo.ts:322-344` (thresholds ≥75 / 50-74 / <50) but the literal value is **`at_risk`** and `GET /studio/students` rows return only `id,name,phone,age,batches` — **no %/tier** (`repo.ts:266-281`). `GET /studio/batches/:id/students` returns `attendance_pct` but **no tier** (`repo.ts:347-390`). Academy `students.tsx` has a **frontend-only** `['All','Music','Dance','Arts','Yoga']` chip bar (`:9`) to drop, an `at_risk` tier pill (`:13`), **no Filters sheet**, and rows with no %/tier badge (`:43-44`). `batches/[id]/index.tsx:99` renders `at_risk` too — and only buckets `≥75` / `<50`, **silently missing the 50–74 band**. Student detail (`students/[id].tsx`) shows attendance % but **no tier badge, no renewal date**; `EnrollmentPeriod.endDate` exists but isn't serialized.
- **🔴 The current % is structurally broken (verified, CRITICAL).** `studio/repo.ts:330-336` computes `present / total` where **both** come from `BatchAttendance` rows — but **only `present:true` rows are ever written** (S3.2 removed manual marking; there is no `recordAbsent` in `src/`). So `total == present` → **every student with any attendance reads 100% = Active**, and a barely-attending student looks perfect. **The denominator must be EXPECTED sessions, not attendance rows.** This is the load-bearing fix of this slice.

## 3. The shared contract (lock first)
```
GET /studio/students?q=&batch_id=&attendance_tier=active|irregular|inactive
  → { items: [{ id, name, phone, age, batches:[…],
                attendance_pct: number|null,         // null = no classes yet ("New")
                tier: 'active'|'irregular'|'inactive'|null }] }

GET /studio/students/:id
  → { student:{…}, enrollments:[{ …, current_period_end: string|null }],   // NEW: renewal date
       tier: 'active'|'irregular'|'inactive'|null, attendance_pct, attendance_history:[…] }
```
- **% = present / EXPECTED (not present/present).** Expected sessions to date = `Batch.sessionsPerMonth` (added S3.4) × months-elapsed since `enrollment.startedAt`, capped at now — `expected = round(sessionsPerMonth × monthsSince(startedAt))`. `present` = count of `present:true` `BatchAttendance` rows for that (user, batch). `pct = expected > 0 ? min(100, round(present/expected × 100)) : null`. *(Alternative denominator: count actual `BatchTiming` occurrences between `startedAt` and now — more precise but heavier; `sessionsPerMonth × months` is the cheap, good-enough version and uses the field we just added.)* **`null` = too new to score** (expected 0) → shown as "New", excluded from tier filters.
- **Per-batch correctness.** When the Filters sheet sends `batch_id`, the % must be **for that batch only** — the present-count query must filter by `batchId` (the current code's tier query omits the batch filter, leaking other batches' attendance into the %). For a multi-batch student with no batch filter, compute per-batch then average (or show per-batch in detail).
- **One tier engine, one vocabulary.** Extract `tierOf(pct: number|null)` (≥75 active / 50–74 irregular / <50 inactive / null→null) into a single helper used by `GET /studio/students`, `GET /studio/batches/:id/students`, **and the batch-hub counts** (`batches/[id]/index.tsx:45-46`, which today drops the 50–74 band) — so the surfaces never disagree. Value is **`irregular`** (never `at_risk`).
- **`attendance_tier` filter** keeps the same param name; only the value `at_risk`→`irregular`. The Filters sheet sends `batch_id` + `attendance_tier`; pills remain a quick attendance filter.

## 4. Component design, per side
- **Backend:** add `lib/attendance-tier.ts` `tierOf(pct)` + `expectedSessions(sessionsPerMonth, startedAt, now)` + `attendancePct(present, expected)` — reuse for all reads. `studio/repo.ts getStudents` — replace the `present/present` calc with `present / expectedSessions` (load `sessionsPerMonth` + `startedAt`), **filter the present-count by `batchId` when set**, return `attendance_pct`+`tier`. Rename the `attendance_tier` enum value + the filter `where`. Apply the same denominator fix to `getBatchStudents` (`repo.ts:347-390`) so the batch roster % is real too. `studio/service.ts getStudent` — serialize the current active period's `endDate` as `current_period_end` (pick via `Enrollment.currentPeriodId`, else the active/grace period with the latest `endDate`) + the student's `tier`. Normalize the no-sessions case to **`null`** everywhere (the current code returns `0` in one path, `null` in another).
- **Academy:** `students.tsx` — delete the category chip bar + its list/batch toggle (dead, see dead-code-registry); pills relabel `at_risk`→**Irregular**; rows render the `tier` badge + `attendance_pct` (or "New"). **BUILD** a Filters bottom sheet (Batch single-select from the academy's batches × Attendance single-select) → sets `batch_id`+`attendance_tier` on `useStudioStudents`. `students/[id].tsx` — add the tier badge + "Plan ends {current_period_end}". `batches/[id]/index.tsx` — `at_risk`→`irregular` label + tier badge on roster preview rows.
